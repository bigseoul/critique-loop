# Plan: critique-with-codex 인터랙티브 라운드 모드 (v0.2.0)

## Context

현재 `critique-with-codex`는 비대화형 자동 N라운드 루프. 두 가지 한계:

1. **Artifact 불변** — Claude가 prior_summary만 만들고 검토 대상은 그대로. Codex 비판이 plan에 실제 반영되지 않음.
2. **사용자 개입 불가** — Codex가 헛소리해도 멈출 수 없음.

v0.2 목표는 **딱 두 가지만**:
- Plan을 versioned mutable artifact로 (v1 → v2 → ...)
- 라운드 끝마다 사용자 승인 체크포인트 (기본 ON)

mid-run auto, feedback history, 복잡한 resume phase 등은 **v0.2.1 이후로 미룬다**. 실제 사용 후 불편하면 추가.

## Round Lifecycle

```
plan-v1.md = 사용자 원본 (init이 저장)
─────────────────────────────────────
[Round N]
  1. Claude → Codex: critique current_plan_path
  2. Codex writes critique-rN.md
  3. Claude reads critique → writes plan-v{N+1}.draft.md
  4. ── CHECKPOINT (interactive=true에서만) ──
     Claude가 짧게 보여줌:
       - Codex 핵심 지적 (severity별 1줄)
       - Claude가 draft에 반영한 변경
       - 보류/거절 + 이유
       - "승인 / 수정요청 / 종료 중 하나로 응답해주세요"
     plan-v{N+1}.draft.md 경로 안내 (전문 보고 싶으면)
     turn 종료 (state.awaiting_user_review = true)
  5. 다음 turn 시작 — Claude가 사용자 응답을 자연어로 해석:
     - **승인 신호** (빈 응답 / "응" / "ok" / "좋아" / "go" 등):
       → cmd_save-plan-version --approve → draft를 plan-v{N+1}.md로 rename
       → current_plan_path / version 갱신, awaiting_user_review=false
     - **종료 신호** ("stop" / "그만" / "종료"):
       → draft 버림. current_plan_path는 plan-vN.md 그대로 (마지막 승인본).
       → Step 6 (합성)으로 직행
     - **수정 요청** (그 외 텍스트):
       → Claude가 draft 다시 수정 → plan-v{N+1}.draft.md 갱신
       → 다시 4번으로 (사용자에게 재표시)
     - **모호** ("좋네", "흠"):
       → 한 번만 재질문. 두 번째 응답은 위 3가지로 분류 강제.
  6. 종료 조건:
     - "종료" → Step 6
     - VERDICT=done → Step 6 (인터랙티브면 "Codex가 done 선언, 동의?" 한 번 묻기)
     - 인터랙티브: max_rounds 없음. 사용자 "종료" 또는 VERDICT=done까지 진행.
     - --auto: N == max_rounds → "더 갈래?" (yes/no)
     - else → 다음 라운드
─────────────────────────────────────
```

**핵심 규칙**:
- `plan-vK.md` = 승인본. `plan-vK.draft.md` = 승인 대기.
- `current_plan_path`는 항상 승인본만 가리킴.
- Codex는 항상 `current_plan_path` (= 최신 승인본)만 봄.
- **stop = draft 버리고 마지막 승인본으로 종료** (승인+종료 원하면 승인 후 "stop").
- 사용자 응답 분류는 Claude가 자연어로 판단. 키워드 매칭 코드 없음.

## Mode

| 모드 | Trigger | 동작 | max_rounds |
|---|---|---|---|
| **interactive (기본)** | `/critique-with-codex` | 매 라운드 끝 체크포인트. 자연어 응답 해석. | **없음**. 사용자가 "stop" 하거나 VERDICT=done 할 때까지 진행. |
| **`--auto`** | `/critique-with-codex --auto` | **체크포인트만 생략**. Claude의 critique → plan 자동 수정은 유지. | **필수** (기본 3, `--rounds N` 오버라이드 1..10). 도달 시 "더?" 묻기. |

**중요**: `--auto`는 v0.1과 동작 다름 (v0.1은 plan 불변, v0.2 `--auto`는 plan을 자동 수정함). 문서에서 "v0.1 동작 보존"이라고 쓰지 않음.

`--rounds N`은 `--auto`와 함께만 의미 있음. 인터랙티브 모드에서 `--rounds`가 와도 무시 (또는 경고). 매 라운드 사용자 승인 게이트가 있으니 안전장치 불필요.

mid-run auto / `auto N` 등은 v0.2 범위 밖 (필요시 v0.2.1).

## Verification

### 자동 단위 테스트 (3개)

1. **`init`이 plan-v1.md를 만들고 current_plan_path 설정** — input_body 주입 후 파일 존재 + state 검증.
2. **`prompt`가 current_plan_path 내용을 사용** — plan-v1.md 수정 후 prompt-r1.md에 그 내용이 들어가는지.
3. **`save-plan-version` draft → approve 동작** — `--draft`로 저장 (state에 draft_plan_path, awaiting_user_review=true) → `--approve`로 rename + state 갱신 확인.

(`auto` 파싱 테스트는 mid-run auto가 빠지면서 불필요.)

### 수동 시나리오 (실제 tmux 셋업)

1. **`--auto` happy path**: `/critique-with-codex --auto --rounds 1 "small plan"` → 1라운드 → plan-v2.md 생성 → 합성 보고서 (chain 표시).
2. **인터랙티브 happy path**: 인터랙티브 호출 → R1 끝 → bullet 요약 표시 → "응" → R2 → "ok" → R3 → R4 진입 (max_rounds 없음) → 사용자 "stop" → 종료.
3. **stop = draft 버림**: R1 끝 → "stop" → plan-v2.draft.md 버려지고 plan-v1.md가 final로.
4. **inner edit loop**: R1 끝 → "X 추가" → draft 갱신/표시 → "Y도" → 갱신 → "ok" → R2 시작 (artifact = 두 번 수정된 draft 승인본).
5. **모호한 응답 재질문**: R1 끝 → "흠" → "승인/수정/종료 중 하나로 응답해주세요" → "ok" → 정상 진행.
6. **resume from awaiting_user_review**: R1 끝 후 세션 종료 → `--resume <rid>` → state 보고 R1 결과 다시 표시.
7. **VERDICT=done 조기 종료**: 인터랙티브에서 "Codex가 done 선언, 동의?" 한 번 묻고 종료.

## Resolved Decisions

1. **기본 = 인터랙티브.** `--auto`는 옵트인.
2. **표시 형식 = bullet 요약** (Codex finding + Claude change + 보류). Plan 전문은 파일 경로 안내.
3. **승인 신호 = Claude의 자연어 판단** (키워드 매칭 코드 없음). 빈 응답, "응", "ok", "좋아" 등 모두 승인. 모호하면 한 번 재질문. SKILL.md에 명시적 안전 키워드(`approve` / `승인`) 안내 추가 — 자연어 판단이 흔들릴 때를 대비한 명확한 지침어.
4. **stop = draft 버리고 마지막 승인본으로 종료.** 승인+종료 원하면 먼저 승인 후 max에서 no.
5. **`--auto` 의미** = 체크포인트 생략. Claude의 plan 자동 수정은 유지 (v0.1과 다름, 문서에 명시).
6. **draft/approved 분리** = `.draft.md` ↔ `.md` rename. `current_plan_path`는 승인본만.
7. **mid-run auto / auto_remaining / user_feedback_history**: v0.2 범위 밖. 필요시 v0.2.1.
8. **synthesize 보고서**: plan version chain + 경로만. 본문 X.
9. **`cmd_save_plan_version` 인터페이스**: `--draft`/`--approve` 두 모드 mutually exclusive.
10. **`max_rounds`는 `--auto` 전용.** 인터랙티브는 무한 루프 (사용자 "stop" / VERDICT=done로만 종료). 매 라운드 승인 게이트가 있어 안전장치 불필요.
